# Data Analyst Telegram Bot
 
Telegram-бот, который анализирует датасеты (CSV/Excel) с помощью LLM-агента.
 
## Возможности
 
-  Автоматический разведочный анализ данных (EDA)
-  Генерация графиков 
-  Обнаружение выбросов методом IQR
-  Бизнес-выводы на основе данных
-  Поддержка пользовательских инструкций для анализа
-  Защита от prompt injection 
-  Поддержка форматов CSV и Excel (.xlsx)
## Технологии
 
- [python-telegram-bot](https://github.com/python-telegram-bot/python-telegram-bot) — Telegram интерфейс
- [Groq API](https://groq.com/) — LLM (llama-3.3-70b-versatile) с tool calling
- [pandas](https://pandas.pydata.org/) — обработка данных
- [matplotlib](https://matplotlib.org/) — визуализация
- [python-dotenv](https://github.com/theskumar/python-dotenv) — управление переменными окружения
## Структура проекта
 
```
├── bot.py            # Telegram-бот, обработка сообщений
├── agent.py          # LLM-агент с tool calling
├── code_executor.py  # Безопасное выполнение Python-кода
├── security.py       # Защита от prompt injection
├── config.py         # Конфигурация и переменные окружения
├── requirements.txt  # Зависимости
├── Procfile          # Для деплоя на Railway
└── .env              # Ключи API 
```

## Как использовать
 
### Базовый анализ
1. Напиши боту `/start`
2. Отправь CSV или Excel файл
3. Получи полный анализ с графиком и выводами
### Анализ с инструкцией
1. Напиши `/analyze найди аномалии в колонке price`
2. Отправь файл
3. Бот выполнит стандартный анализ + твой запрос
### Команды
 
| Команда | Описание |
|--------|----------|
| `/start` | Приветствие и инструкция |
| `/analyze [текст]` | Задать инструкцию для анализа |
| `/help` | Справка по командам |
 
## Что выдаёт бот
 
1. **Описание датасета** — форма, типы колонок, пропуски, первые строки, статистика
2. **График** — распределения числовых колонок или тепловая карта корреляций
3. **Анализ** — выбросы, корреляции или результат твоего запроса
4. **Бизнес-выводы** — минимум 3 вывода на основе данных
## Безопасность
 
- **Двухуровневая защита от prompt injection:** проверка по паттернам + LLM-классификатор
- **Проверка содержимого датасета:** сканирование текстовых ячеек на инъекции
- **Ограниченное окружение выполнения кода:** запрещены `os`, `subprocess`, `eval`, `exec` и другие операции
- **Проверка кода перед выполнением:** весь код, сгенерированный LLM, проверяется перед запуском

## Ограничения
 
- Поддерживаются только форматы `.csv` и `.xlsx`
- Время анализа — 15–40 секунд в зависимости от датасета
